# cw-brainstorming — Minimal-capture creative writing brainstorm notes

> Capture story brainstorming faithfully — exactly what was said, nothing invented. Uses a 3-tag source system to keep AI suggestions clearly separate from author decisions.

[![OpenClaw Skill](https://img.shields.io/badge/OpenClaw-Skill-blueviolet)](https://github.com/NachaFromMars)

## Overview
cw-brainstorming captures creative writing brainstorming sessions in working-note format that preserves creative freedom. Its core rule is minimal capture: record only what the user explicitly stated, mark AI suggestions with `<AI>` tags, and mark unclear items with `[?]`. Vagueness in the source is preserved — gaps in the notes are features, not bugs, leaving room for later creative decisions.

## Features
- **Minimal capture** — only what the user said; no elaboration
- **3-tag source system** — untagged = user said it; `<AI>` = AI suggestion; `[?]` = unclear
- **Types handled** — story/plot, chapter structure & beats, worldbuilding & lore, character development, timeline & continuity
- **Reference files** — `chapter-planning.md`, `worldbuilding.md`, `character-development.md`, `continuity-timeline.md`

## Usage / Quick Start
Use during any story exploration conversation. The skill produces skeletal working notes from the session.
